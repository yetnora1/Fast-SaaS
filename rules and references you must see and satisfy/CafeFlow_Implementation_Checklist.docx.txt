CafeFlow SaaS
Implementation Checklist
━━━━━━━━━━━━━━━━━━━━━━━━
Every role goal  →  DB table  →  API route  →  UI component
8 Roles  •  4 Build Phases  •  Production-Ready

P1  Foundation (Wk 1-3)
P2  Core Flow (Wk 4-6)
P3  Payments (Wk 7-9)
P4  Polish (Wk 10+)

Ph = Build Phase. Each goal is tagged P1–P4 so you can build in priority order. Complete P1 first — it is the foundation every other phase depends on.

Build Phases — What to build when
Build in strict phase order. P1 is the skeleton — nothing else works without it. Each phase produces a testable milestone.

P1 — FOUNDATION

PostgreSQL schema + Prisma migrations for all core tables
NextAuth.js v5 — JWT with role + tenant_id in session token
Tenant isolation middleware — every query scoped to tenant_id
Row Level Security (RLS) on PostgreSQL for tenant data isolation
SaaS Owner admin panel + approval workflow (most critical)
Subscription state machine: TRIAL → PENDING → ACTIVE → EXPIRED
Server-side subscription gate middleware (not just UI hide)
Menu CRUD — minimum viable menu before any orders can flow
Staff invite + role assignment
PIN auth for POS tablet login
Allergy alert framework in order_items schema

P2 — CORE ORDER FLOW

Firebase Firestore project setup + real-time listeners
Waiter floor map + order builder (biggest UI piece)
Order split logic: drinks → barista_kds, food → kitchen_kds
Barista KDS board + Kitchen KDS board (both Firebase real-time)
Order state machine: DRAFT→SUBMITTED→PREPARING→READY→DELIVERED
Cashier POS: bill queue + cash payment + basic receipt
Shift open/close + cash reconciliation
Manager live order board + void (PIN) + refund approve
Firebase Cloud Messaging (FCM) push notifications
Table status map for waiter and manager
Low stock alerts + restock request flow

P3 — PAYMENTS & INVENTORY

Telebirr API integration + webhook endpoint
CBE Birr API integration + webhook endpoint
Ethiopian ERCA-compliant receipt (VAT 15% breakdown)
ESC/POS thermal printer integration (escpos npm)
Full inventory module: receive, adjust, PO, supplier CRUD
Stock count audit wizard
Stockout forecast algorithm (PLpgSQL stored proc)
Multi-branch comparison dashboard
Shift PDF report generation (pdf-lib or puppeteer)
QR self-ordering customer flow
Staff scheduling calendar

P4 — POLISH & LAUNCH

Service Worker (PWA) offline mode + IndexedDB sync
Amharic (AM) language toggle for all UI text
Ethiopian calendar support (Ge'ez) for scheduling
Telegram bot integration for daily owner summaries
Analytics: VAT report, payroll summary, consumption report
Load testing (k6) — 50 orders/min, 100 concurrent users
Security audit: penetration test, rate limiting, input sanitization
Sentry error tracking + Vercel Analytics
Staff performance leaderboard + ratings
Daily briefing card (AI-style insight summary)
Full audit log viewer for SaaS Owner
