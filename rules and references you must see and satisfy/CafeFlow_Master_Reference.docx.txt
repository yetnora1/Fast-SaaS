Complete Order Lifecycle
This is the spine of the system. Every role connects through this flow. Build this in P2 and everything else plugs into it.

DINE-IN ORDER — FULL ROLE SEQUENCE

Step
What happens
1. Waiter seats customer
Taps table on floor map → table turns orange (occupied)
2. Waiter builds order
Browses menu, selects items, chooses modifiers, adds notes
3. Waiter sends order
Taps Send → server splits: drinks → barista_kds, food → kitchen_kds
4. Barista receives
Drink cards appear on KDS → audio alert → tap Start Preparing
5. Kitchen receives
Food cards appear on KDS → tap Accept → Start cooking
6. Barista completes
Taps Ready on each drink → Order Ready → FCM push to Waiter
7. Kitchen completes
Taps Ready → checks bar sync → if both ready → FCM to Waiter
8. Waiter delivers
Receives notification → delivers items → taps Delivered per item
9. Bill requested
Customer ready → Waiter taps Request Bill → appears in Cashier queue
10. Cashier processes
Opens bill → selects payment method → confirms payment
11. Receipt generated
ERCA-compliant receipt printed/sent → table freed → Waiter notified
12. System updates
Inventory decremented, order logged, shift totals updated, KPIs refresh

ORDER STATE MACHINE

State
Meaning
DRAFT
Waiter is building the order — not yet sent
SUBMITTED
Sent to system — being routed to KDS
IN_PREPARATION
Barista/Kitchen actively working on items
PARTIALLY_READY
Some items done, others still in progress
READY
All items prepared, waiting for delivery
DELIVERED
All items confirmed delivered by Waiter
BILL_REQUESTED
Bill request sent to Cashier queue
PAYMENT_PENDING
Cashier opened bill, awaiting payment confirmation
COMPLETED
Paid, receipt issued, table freed — final state
VOIDED
Manager cancelled pre-payment (with PIN + reason)
REFUNDED
Refund processed post-payment by Cashier

SUBSCRIPTION GATE STATE MACHINE

State
Condition and Access Level
TRIAL
0–7 days after signup — full access, countdown banner shown
PENDING
Receipt uploaded, awaiting SaaS Owner approval — read-only access
ACTIVE
Approved — full access for 180 days
WARNING
T-7 days before expiry — banner + modal on next login, full access
GRACE
Expired — 3-day grace period, degraded view-only access
EXPIRED
Grace over — full lockout, gate modal on every route
SUSPENDED
Manually suspended by SaaS Owner — immediate lockout, no modal
